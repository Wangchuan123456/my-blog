## 前言

今天是跟着黑马教程学习前端的第一天。说实话，打开视频之前，我对"前端"的认知就是"写网页的人"。现在学完了 HTML 基础部分，来整理一下学到的东西。

## 什么是 HTML？

HTML 的全称是 **HyperText Markup Language**（超文本标记语言）。

老师强调的第一句话就是：**HTML 不是编程语言，而是一种标记语言**。它不负责逻辑（那是 JavaScript 的事），也不负责样式（那是 CSS 的事），它只做一件事——**告诉浏览器：这是什么**。

比如：

```html
<h1>这是标题</h1>
<p>这是段落</p>
```

浏览器看到 `<h1>` 就知道"哦，这是个一级标题"，看到 `<p>` 就知道"这是段落"。HTML 就是网页的**骨架**。

## HTML 文档的基本结构

一个完整的 HTML 文档长这样：

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8" />
  <title>我的第一个页面</title>
</head>
<body>
  这里写页面内容
</body>
</html>
```

各部分的作用：

| 部分 | 作用 |
|------|------|
| `<!DOCTYPE html>` | 文档类型声明，告诉浏览器用 HTML5 标准来解析 |
| `<html>` | 根标签，所有内容都写在它里面 |
| `<head>` | 头部，放**不直接显示**的配置信息（编码、标题、样式等） |
| `<body>` | 身体，放**页面上能看到**的所有内容 |

> **踩坑记录：** 我第一次写的时候把 `<title>` 写到了 `<body>` 里，结果标题出现在了页面左上角而不是浏览器标签栏上。查了才知道：`<title>` 必须放在 `<head>` 里。

## 标签关系：并列 vs 包含

这是老师讲的一个非常重要的概念——**标签之间的关系只有两种**。

### 1. 并列关系（兄弟关系）

```html
<head></head>
<body></body>
```

`<head>` 和 `<body>` 谁都不在谁里面，它们是同一级的，叫**并列**。

### 2. 包含关系（父子关系）

```html
<html>
  <head>
    <title>我的页面</title>
  </head>
</html>
```

`<title>` 在 `<head>` 里面，`<head>` 在 `<html>` 里面——这叫**包含（嵌套）**。

**我的理解方法：** 把 HTML 想象成一个文件夹：
- `html` 是根目录
- `head` 和 `body` 是根目录下的两个同级文件夹
- `title` 是 `head` 文件夹里的一个文件

这样一想，标签的嵌套结构就清晰了。

## 标题标签 `<h1> ~ <h6>`

HTML 提供了 6 级标题，从 `<h1>`（最大）到 `<h6>`（最小）：

```html
<h1>一级标题 — 文章大标题</h1>
<h2>二级标题 — 章节标题</h2>
<h3>三级标题 — 小节标题</h3>
<h4>四级标题</h4>
<h5>五级标题</h5>
<h6>六级标题</h6>
```

**重要原则：**
- 一个页面**最好只用一个** `<h1>`（给文章主标题）
- 按层级使用，不要跳级（`<h1>` → `<h2>` → `<h3>`，不要 `<h1>` 直接到 `<h3>`）
- 标题自带加粗和上下间距，这是浏览器默认样式

> 老师说：搜索引擎会把 `<h1>` 看作页面最重要的内容，所以 SEO 优化时 `<h1>` 要放核心关键词。

## 段落标签 `<p>`

```html
<p>这是一个段落。段落前后会自动换行，并且上下有间距。</p>
<p>这是第二个段落。浏览器会自动处理段落之间的空白。</p>
```

段落标签的几个特点：
- 块级元素，**独占一行**
- 前后有默认的 `margin`
- 多个空格和换行会被合并成一个空格

```html
<!-- 下面两种写法，浏览器显示效果完全一样 -->
<p>hello     world</p>
<p>hello world</p>
```

## 语义化的概念（重点）

老师花了不少时间讲**语义化**，我觉得这是今天学到最重要的概念。

**语义化的核心思想：** 用最合适的标签去表达内容的含义，而不是只看它长什么样。

### 反例：非语义化

```html
<!-- 标题看起来像标题，但实际上是段落加粗放大 -->
<p style="font-size: 32px; font-weight: bold;">这是一个"假标题"</p>
```

### 正例：语义化

```html
<h1>这是一个真标题</h1>
```

### 为什么语义化重要？

1. **SEO 友好**：搜索引擎根据标签判断内容权重
2. **无障碍访问**：屏幕朗读器可以正确解读页面结构
3. **代码可维护性**：别人看你的代码，一眼就知道哪里是标题、哪里是导航、哪里是主要内容
4. **默认样式**：浏览器给不同标签提供了合理的默认样式

老师举了一个例子：你可以用 `<div>` 和 CSS 模拟出任何标签的外观，但**外观可以模仿，含义无法替代**。

## 今天练习的代码

学完后我照着视频写了一个小页面：

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8" />
  <title>我的学习笔记</title>
</head>
<body>
  <h1>前端学习日志</h1>
  <h2>第一天：HTML 基础</h2>
  <p>今天我学习了 HTML 的基本结构、标签的并列和包含关系、标题和段落标签，还有语义化的概念。</p>
  <h2>第二天计划</h2>
  <p>准备学习文本格式化标签、图片标签和链接标签。</p>
</body>
</html>
```

这是我写的第一个真正的 HTML 页面。虽然很简单，但看到它在浏览器里渲染出来的那一刻，还是有那么一点成就感 😄
