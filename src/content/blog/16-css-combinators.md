## 前言

前面学了四种**基础选择器**（类型、类、ID、通配符），它们都是单独选中元素。

但很多时候，我们需要**根据元素之间的关系**来选中目标——比如"导航栏里的所有链接"、"div 的直接子元素"、"h2 后面的段落"。

这就用到**关系选择器**（也叫组合选择器 Combinators）。

CSS 有四种关系选择器：

| 名称 | 符号 | 含义 | 示例 |
|------|:----:|------|------|
| **后代选择器** | 空格 | 选中所有后代（子/孙/曾孙…） | `.nav a` |
| **子代选择器** | `>` | 只选直接子元素 | `div > span` |
| **邻接兄弟选择器** | `+` | 选中紧跟在后面的第一个兄弟 | `h2 + p` |
| **通用兄弟选择器** | `~` | 选中后面的所有兄弟 | `h2 ~ p` |

## 一、后代选择器（Descendant Selector）

用**空格**连接，选中父元素**内部所有层级的**匹配元素。

```css
.nav a {
  color: green;
}
```

```html
<ul class="nav">
  <li><a href="#">链接 1</a></li>        <!-- ✅ 选中 -->
  <li>
    <span><a href="#">链接 2</a></span>  <!-- ✅ 选中（即使嵌套更深） -->
  </li>
</ul>
```

### 特点

| 项目 | 说明 |
|------|------|
| 符号 | 空格 `父 子` |
| 层级 | **不限层级**——子、孙、曾孙都会选中 |
| 适用场景 | 选中某个容器内的所有同类元素 |

### 示例

```css
.footer a {
  color: red;
}
```

`.footer` 里的所有 `<a>` 都变红，不管嵌套多深。

## 二、子代选择器（Child Selector）

用 `>` 连接，**只选直接子元素**，不选孙子。

```css
div > span {
  color: blue;
}

p > span {
  color: pink;
}
```

```html
<div>
  <span>我是儿子</span>       <!-- ✅ div > span 选中，蓝色 -->
  <p>
    <span>我是孙子</span>     <!-- ❌ div > span 不选（孙子） -->
                              <!-- ✅ p > span 选中，粉色 -->
  </p>
</div>
```

### 特点

| 项目 | 说明 |
|------|------|
| 符号 | `>` 直接子元素 |
| 层级 | **仅一层**——只选直接子元素 |
| 适用场景 | 只想选中直接子元素，排除更深层嵌套 |

### 后代 vs 子代

```
后代选择器（空格）：div span → 选中所有 span，不管在 div 里面多深
子代选择器（>）：   div > span → 只选 div 的直接子元素 span，不选孙子
```

## 三、邻接兄弟选择器（Adjacent Sibling Selector）

用 `+` 连接，选中**紧跟在指定元素后面的第一个**兄弟元素。

```css
h2 + p {
  color: red;
}
```

```html
<h2>我是标题</h2>
<p>我是紧跟在 h2 后面的 p</p>    <!-- ✅ h2 + p 选中，红色 -->
<p>我是再后面的 p</p>            <!-- ❌ 不选中 -->
```

### 特点

| 项目 | 说明 |
|------|------|
| 符号 | `+` 紧邻后面 |
| 选中数量 | **仅一个**——紧跟在后面的第一个兄弟 |
| 适用场景 | 标题后的第一段特殊样式 |

## 四、通用兄弟选择器（General Sibling Selector）

用 `~` 连接，选中**指定元素后面的所有**兄弟元素。

```css
h2 ~ p {
  color: red;
}
```

```html
<p>h2 前面的 p</p>              <!-- ❌ 不选中 -->
<h2>我是标题</h2>
<p>h2 后面的 p 1</p>            <!-- ✅ h2 ~ p 选中，红色 -->
<p>h2 后面的 p 2</p>            <!-- ✅ 选中，红色 -->
```

### 特点

| 项目 | 说明 |
|------|------|
| 符号 | `~` 后面所有 |
| 选中数量 | **全部**——后面的所有兄弟都选中 |
| 适用场景 | 标题后所有段落统一样式 |

### + 与 ~ 的区别

```
h2 + p：只选 h2 后面紧跟着的 第一个 p
h2 ~ p：选 h2 后面 所有的 p
```

## 五、四种关系选择器对比

| 名称 | 符号 | 选中范围 | 说人话 |
|------|:----:|---------|--------|
| 后代 | 空格 | 内部所有层级 | 容器里的所有 xxx |
| 子代 | `>` | 仅直接子元素 | 我的直接儿子 |
| 邻接兄弟 | `+` | 紧邻的下一个 | 我后面的第一个兄弟 |
| 通用兄弟 | `~` | 后面的所有 | 我后面的所有兄弟 |

### 可以组合使用

```css
/* 导航栏里 li 中的直接 a 标签 */
.nav > li > a { ... }

/* 主体内容里 h2 后面的第一个 p */
.main h2 + p { ... }
```

## 小结

1. **空格（后代）** — 选所有后代，最常用
2. **`>`（子代）** — 只选直接子元素，精确控制层级
3. **`+`（邻接兄弟）** — 选紧跟在后面的第一个兄弟
4. **`~`（通用兄弟）** — 选后面的所有兄弟

关系选择器让 CSS 变得更加精准和强大——不靠加 class，靠**元素之间的位置关系**就能选中目标。
