## 前言

CSS（Cascading Style Sheets）是用来给 HTML **设置样式**的语言——颜色、大小、布局、动画等都是 CSS 说了算。

把 CSS 应用到 HTML 上有**三种方式**：

1. **行内样式** — 直接在 HTML 标签的 `style` 属性里写
2. **内部样式** — 在 HTML 的 `<style>` 标签里写
3. **外部样式** — 写在独立的 `.css` 文件里，用 `<link>` 引入

下面逐个来看。

## 一、行内样式（Inline）

直接在 HTML 标签的 `style` 属性中写 CSS。

```html
<div style="color: blue; font-size: 20px;">
  我喜欢蓝色
</div>
```

### 特点

| 项目 | 说明 |
|------|------|
| 写法 | `style="属性名: 属性值;"` |
| 作用范围 | **仅当前这一个标签** |
| 优先级 | **最高**（会覆盖其他方式的同属性样式） |
| 适用场景 | 测试、临时调整、JS 动态修改样式 |

### 优缺点

**优点：**
- 最直接，想给哪个标签设样式就在哪个标签上写
- 优先级最高，可以用来覆盖其他样式

**缺点：**
- 不能复用——同样的样式要在每个标签上重复写
- HTML 和 CSS 混在一起，代码难维护
- 不适合写大量样式

## 二、内部样式（Internal）

在 HTML 的 `<head>` 中用 `<style>` 标签包裹 CSS。

```html
<!DOCTYPE html>
<html>
<head>
  <style>
    strong {
      color: red;
      font-weight: bold;
    }
    p {
      font-size: 16px;
    }
  </style>
</head>
<body>
  <strong>我喜欢红色</strong>
  <p>这是一段文字</p>
</body>
</html>
```

### 特点

| 项目 | 说明 |
|------|------|
| 写法 | `<style>` 标签放在 `<head>` 中 |
| 作用范围 | **当前整个 HTML 页面** |
| 优先级 | 仅次于行内样式 |
| 适用场景 | 单页面、小项目、学习练习 |

### 优缺点

**优点：**
- 样式可以复用——页面内所有相同标签都会生效
- HTML 和 CSS 在同一个文件中，方便查看
- 适合小页面或学习练习

**缺点：**
- 不能跨页面复用——每个 HTML 文件都要写一遍
- 文件内部仍然混杂，页面大了不好管理

## 三、外部样式（External）

把 CSS 单独写在一个 `.css` 文件中，然后用 `<link>` 标签引入 HTML。

**style.css：**
```css
p {
  color: green;
  line-height: 1.8;
}

strong {
  color: red;
}
```

**index.html：**
```html
<!DOCTYPE html>
<html>
<head>
  <link rel="stylesheet" href="./style.css">
</head>
<body>
  <p>我喜欢绿色</p>
  <strong>我喜欢红色</strong>
</body>
</html>
```

### 特点

| 项目 | 说明 |
|------|------|
| 写法 | `<link rel="stylesheet" href="文件路径">` |
| 作用范围 | **所有引入了该 CSS 文件的页面** |
| 优先级 | 最低（但可利用选择器权重覆盖） |
| 适用场景 | 生产项目、多页面网站 |

### 优缺点

**优点：**
- **完全复用**——一个 CSS 文件可以被多个 HTML 页面共用
- **彻底分离**——HTML 负责结构，CSS 负责样式，职责清晰
- **便于维护**——改一个 CSS 文件就能影响所有页面
- **缓存优化**——浏览器可以缓存 CSS 文件，加快加载速度

**缺点：**
- 需要额外的 `<link>` 引入步骤
- 首次加载需要多一个 HTTP 请求（但缓存后就不存在了）

## 四、三种方式对比

| 对比项 | 行内样式 | 内部样式 | 外部样式 |
|--------|---------|---------|---------|
| 写法位置 | 标签 `style` 属性 | `<head>` 中的 `<style>` | 独立 `.css` 文件 |
| 复用范围 | 仅当前标签 | 当前页面 | 所有引入的页面 |
| 优先级 | 最高 | 中间 | 最低 |
| 适用项目 | 临时测试 | 小页面/学习 | 生产项目/多页面 |
| HTML/CSS 分离 | ❌ 混合 | ❌ 同文件分离 | ✅ 完全分离 |
| 维护性 | 差 | 一般 | 好 |

### 优先级规则

当同一个元素有多个样式冲突时，**就近原则**——离元素越近的样式优先级越高：

```
行内样式 > 内部样式 > 外部样式
```

但注意，这只是**同权重下**的规则。选择器的权重（如 id > class > 标签）优先级高于样式表来源。

## 小结

1. **行内样式**写在标签 `style` 属性中——优先级最高，但不推荐大量使用
2. **内部样式**写在 `<style>` 标签中——适合单页面
3. **外部样式**写在 `.css` 文件中，用 `<link>` 引入——**最推荐的方式**，适合实际项目

实际开发中，**绝大多数场景都用外部样式**。行内样式和内部样式只在特定场景下使用。
