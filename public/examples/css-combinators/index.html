🔗 关系选择器综合演示

四种关系选择器——空格 后代 、> 子代 、+ 邻接兄弟 、~ 通用兄弟。

1️⃣ 后代选择器 空格

选中父元素内部所有层级的匹配元素：

.nav a — 导航内的所有链接变成绿色：

  • 首页
  • 文章
  • 更多 
/* 后代选择器：选中 .nav 内所有 a，不限层级 */
.demo-nav a {
  color: #27ae60;
  font-weight: bold;
}

2️⃣ 子代选择器 >

只选直接子元素，不选孙子：

div > span — 只选 div 的直接子元素 span：

🔵 我是儿子（被选中 → 蓝色）

🔴 我是孙子（不被 div > span 选中，但被 p > span 选中 → 红色） 

/* 子代选择器：只选直接子元素 */
.demo-child > span { color: #2980b9; }   /* 只选 div 里的直接 span */
.demo-child p > span { color: #e74c3c; } /* 选 p 里的直接 span */

3️⃣ 邻接兄弟选择器 +

只选紧跟在指定元素后面的第一个兄弟：

h3 + p — h3 后面紧跟着的第一个 p 变红：

这是 h3 前面的 p（不变）

📌 我是 h3 标题

✅ 我是 h3 后面的第一个 p，被 + 选中！

❌ 我是第二个 p，+ 不选中我

❌ 第三个 p，也不选中

/* 邻接兄弟：h3 后面紧跟着的 p */
.demo-adjacent h3 + p {
  color: #e74c3c;
  font-weight: bold;
}

4️⃣ 通用兄弟选择器 ~

选中指定元素后面的所有兄弟：

h3 ~ p — h3 后面的所有 p 变红：

这是 h3 前面的 p（不变）

📌 我是 h3 标题

✅ 后面的第一个 p，被 ~ 选中

✅ 第二个 p，也被选中

✅ 第三个 p，全部选中

/* 通用兄弟：h3 后面的所有 p */
.demo-general h3 ~ p {
  color: #e74c3c;
  font-weight: bold;
}

5️⃣ + 与 ~ 的区别

选择器 符号 选中范围
后代 空格 容器内所有层级的匹配元素
子代 > 仅直接子元素
邻接兄弟 + 紧跟在后面的第一个兄弟
通用兄弟 ~ 后面的所有兄弟
💡 记忆技巧：
空格 → 容器里的所有 xxx（不限层级）
> → 我的直接儿子（仅一层）
+ → 我后面的第一个兄弟
~ → 我后面的所有兄弟
